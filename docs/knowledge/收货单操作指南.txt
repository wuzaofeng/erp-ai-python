# 收货单操作指南

## 功能说明

收货单用于记录采购物料的实际入库情况，必须关联已审批的采购订单才能创建。
收货完成后系统自动更新采购订单的已收货数量和待收货数量。

## 主表：T_PUD_Receive_Order

| 字段名 | 中文名 | 说明 |
|--------|--------|------|
| fBillNo | 单据编号 | 系统自动生成，格式：RCV+年月日+流水号 |
| fStatus | 单据状态 | 0=草稿, 1=已提交, 2=已审批, 3=已作废 |
| fPurchaseOrderNo | 采购订单编号 | 关联采购订单 |
| fSupplierCode | 供应商编码 | 从采购订单带出 |
| fReceiveDate | 收货日期 | 默认当天 |
| fWarehouseCode | 入库仓库编码 | 必填 |
| fStorageCode | 库位编码 | 可选 |
| fReceiverCode | 收货人编码 | 默认当前登录用户 |

## 明细表：T_PUD_Receive_Order_Item

| 字段名 | 中文名 | 说明 |
|--------|--------|------|
| fMaterialCode | 物料编码 | 从采购订单明细带出 |
| fOrderQty | 订单数量 | 采购订单数量 |
| fReceiveQty | 本次收货数量 | 必填，不能超过待收数量 |
| fQualifiedQty | 合格数量 | 质检后填写 |
| fUnqualifiedQty | 不合格数量 | 质检后填写 |
| fUnitCost | 入库单价 | 从采购订单单价带出 |

## 操作步骤

### 第一步：新建收货单
1. 进入【采购管理】→【收货单】
2. 点击【新增】按钮
3. 选择关联的采购订单编号
4. 系统自动带出供应商、物料信息

### 第二步：填写收货明细
1. 核对物料信息是否正确
2. 填写【本次收货数量】，不得超过待收数量
3. 选择【入库仓库】（必填）
4. 如有质检要求，填写合格数量和不合格数量

### 第三步：提交审批
1. 确认所有信息无误
2. 点击【提交】，进入审批流
3. 审批通过后库存自动增加
4. 采购订单的已收货数量同步更新

## 状态说明

- 草稿（0）：可修改，未影响库存
- 已提交（1）：等待审批，不可修改
- 已审批（2）：库存已入账，不可修改
- 已作废（3）：不影响库存统计

## 常见问题

### 收货数量超过订单数量怎么办？
系统会提示错误，收货数量不允许超过采购订单的待收数量（fRemainingQty）。
如需超收，须先修改采购订单数量并重新审批。

### 收货单提交后发现填错了怎么办？
已提交状态不可修改，需联系审批人退回。
审批通过后（已入库）只能做退货处理，不能直接修改。

### 如何查看某采购订单的收货进度？
在采购订单列表中查看 fReceivedQty（已收货数量）和 fRemainingQty（待收数量）。
或在收货单中按采购订单编号筛选查看所有收货记录。
